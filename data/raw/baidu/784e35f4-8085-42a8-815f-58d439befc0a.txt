工作职责
1、复杂环境下的工程交付
2、深度参与数据平台在客户侧的私有化部署、适配与运行保障，处理复杂基础设施、网络与历史系统约束
3、在资源受限、环境差异大的条件下，推动系统成功上线并长期稳定运行
4、代码级问题解决与能力增强
5、针对客户现场暴露的问题，进行源码级分析、修复与优化，而非仅停留在配置或脚本层
6、在必要情况下，为特定场景补齐平台能力（功能、性能、稳定性或可运维性），并推动能力平台化
7、平台工程质量与架构改进
8、参与平台关键模块的设计与改进，对系统可扩展性、稳定性、性能与可维护性负责
9、推动私有化与生产环境下的工程标准建设，包括部署、升级、回滚、监控与故障恢复
10、研发与交付的双向桥梁
11、将客户侧复杂问题抽象为工程问题，与核心研发团队协同推动平台级修复与演进
12、对平台在真实生产环境中的表现负责，而不仅是功能交付完成
13、关键节点技术兜底
14、在高风险或关键阶段，作为技术兜底角色参与问题处理与决策
15、接受阶段性出差，参与客户现场关键交付与疑难问题处理

任职要求
1、工程背景
2、本科及以上学历，计算机相关专业优先；不设年龄限制，更关注工程能力与成熟度
3、3年及以上后端/平台/分布式系统相关研发经验
4、有ToB私有化系统、平台级产品或复杂生产系统经验者优先
5、核心工程能力
6、精通至少一门主流编程语言（Java / Go / C++ / Python 等），具备扎实的设计、调试与重构能力
7、熟悉 Linux 系统，能够进行系统级问题定位（CPU、内存、IO、网络、JVM 等）
8、具备阅读、理解并修改复杂系统源码的能力，能够从根因分析到代码级修复形成闭环
9、分布式与数据平台理解
10、对分布式系统有系统性认知，包括计算、存储、调度、容错与扩展性设计
11、熟悉数据平台或大数据技术栈（如 Spark / Flink / SQL 引擎 / 调度与资源管理等）
12、理解数据平台在生产环境中的关键挑战：性能、稳定性、数据一致性与运维复杂度
13、沟通与现场能力
14、能进行清晰、专业的技术沟通，与客户及内部团队建立技术信任
15、能接受一定频率的出差与现场工作节奏
16、能在需求不清晰、环境不标准的情况下独立推进技术落地
17、以工程手段解决问题，而非依赖流程或产品妥协
18、强Ownership意识，对系统长期稳定运行负责
